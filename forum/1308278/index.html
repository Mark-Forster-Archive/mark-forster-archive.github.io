  • I've thought about this some. I think whatever you did for DWM or AF1 may work. Mostly, the two-column aspect of SAF isn't very important. What matters is that you can add tasks in two places: the current page and the last page, and that you have a way to decide a page is full, or you need to start a new page. And you need the ability to add tasks on any page.

    So just have multiple groups for your tasks, and always add tasks to either the current group or the last one. In a calendar system(but doing SAF), add to the active date or the newest open date. Just ignore the dates.

    And remember to stop adding new things to the last group when it's "half-full".

  • Sorry for the late reply.

    I have my pad set as fixed length (25 items). Whenever I notice that a new page has been started I go back to the previous page and add a line ('=======') by tapping the '+' row at the bottom which basically draws the line between the left and right colums of that page. When I then want to add a task to the second column, I tap the '+' row again, which will add a task to the curren page *provided* that you have this option enabled in the application settings.

    If tapping the '+' row adds a task to the last page instead of the current page, that option may be disabled or it may be a bug. It wasn't working for me at some point and then it suddenly started working again. I haven't been able to reproduce it.

Implementing Autofocus digitally, with Things

  • Like a number of other people, my preference is for the digital, and as a Mac user I'm a big fan of Things as task management software. (Things has an iPhone app, too.) I've been using Things very successfully with DIT for quite a while now, and so I was a bit disappointed when I couldn't see right away how to use it as a vehicle for Autofocus. (My disappointment was sharpened by the fact that Things 1.0 was finally released this weekend, so that I've just paid actual money for it....)

    This morning, though, it struck me that there's a perfectly simple way of making the combination work. The trick I'm using is just to create a new project to correspond to each page of the paper notebook I'd otherwise be using. (I'm using the term 'project' here the way it's used in Things, not the way it's used in, e.g., DIT.) So I now have projects entitled "Page 1", "Page 2", etc., and I can cycle through those projects in just the way I'd cycle through the pages of a paper notebook. What's more, this buys me many of the advantages Frank describes for his electronic implementation in an earlier post:

    http://markforster.squarespace.com/forum/post/622809

    In Things, the equivalent of dismissing the remaining tasks on a page will be to render the relevant project ("Page 7", e.g.) inactive, which archives it but removes it from the list in the sidebar, so that you're no longer tempted to cycle through it. It's also quick and easy to make a copy of a task and move it to the last page before checking it off, when it's an unfinished task or a recurring one.

    Finally, there are also a couple of ways to solve what seemed at first sight to be the main problem: enforcing a limit on the number of tasks per page. One option would simply be to select the "Projects" Focus in the sidebar, which then brings up a list of all one's projects (pages) with an item count for each project. You could use that view to check that you haven't added too many items to the last page. If you have, simply create a new page, go back to the overburdened page, select the extra items over your limit (of 30 per page, say), and drag them to the new last page.

    That's a bit clunky, of course (though not as bad as it sounds, perhaps), and involves a bit of work that has no counterpart if you're just using a paper notebook. But there's another, more elegant solution if the screen you're working on is big enough: set the size of your Things window so that it's just big enough to contain 30 items (say). Then you get a straightforward visual cue when it's time to start a new page.

    This works for me because most of the time I'm working with a cinema display attached to my laptop. Admittedly, though, the elegant solution isn't perfect when working from a normal laptop screen. When I unplug, for example, my item-per-page limit, if I want to employ this solution, is 21, or 22 if I move the OS X dock to one side of my screen, or 23 if I hide the Things toolbar, or 24 if I do both...tantalisingly close to the recommended 25-35 lines per page range. I wouldn't like to have my dock on the side of my screen, but perhaps reducing the size of a page to 21 or 22 items per page wouldn't interfere with the smooth running of Autofocus; I don't know.

    Of course, there is a real obstacle to using the elegant solution if you're doing most of the inputting on your iPhone, where you can only see 7-8 items at a time. But the clunkier solution is still an option there....

    Anyway, hope this is useful for someone.

  • I'm another GTD to Autofocus convert and I use Things Touch exclusively (I don't have a Mac). Here's my setup in case anyone is interested.

    Since the screen only allows me to view 8 lines at a time and I don't want to continually remember how many items I have on a page the first thing I do in the morning is create a Project with the current date as the title (i.e., 2/17/2009). This is the "active" page for the day. Anything that I work on that doesn't get completed gets moved here (i.e., if I work on something from my 2/15 page and decide I'm good for now, I move it to the end of 2/17 simply by pressing the little arrow button at the bottom, selecting the task and then selecting the 2/17 page).

    I deviate a little bit from AF to suit my own needs: to handle things that have to be done on a specific day, I file things in the "Scheduled" list and make use of the "Due date" field when appropriate. I also have one tag called "Repeat" that I use for things that get done on some sort of repeated timeframe (i.e., take out the trash, make lunches, annoy the neighbors). At the start of the day, I can go to my Logbook, filter on the Repeat tag and add them all to my new active page and then schedule or do whatever with them.

    When a page is ready to be dismissed I just move the project into the "Someday" bucket and set myself a reminder to go back and check it in a week or so.

    Works for me really well, so I thought I'd share.

Alternative digital interface

  • Hello everyone,

    I recently had the idea that you don't really need a list at all for using autofocus - at least not a traditional list which shows several items at once.

    The autofocus instructions tell us to always concentrate on one task and one task alone, make a decision about it, then consider the next task. An interface that shows only the current task would bear less risk of not following the autofocus process by aimlessly scanning the whole page.

    So I thought about implementing autofocus with index cards, a index card corresponds not to a page, but to a single task, and a page is a deck of cards, and the whole "list" consists of several decks. That approach is a little bulky in paper form, but digitally I think it could really shine.

    I created a little demo here: http://wiki.autofocus.cc/dojo/

    It contains five dummy task cards. If you click "Next Task", you shuffle through the deck. If you leave the last card, you get back to the first. If you click "Mark as done", the card get's removed from the deck.

    What do you think of this approach?

    I think it would enable a far more accurate implementation of the autofocus process, you could have a skimming face at the beginning, where you can't do anything but shuffle through the deck. After you are through, you go to the first "doing" pass. There you get asked for each card "Do you feel like working on this?" if you answer yes, you work on it and can then choose if you're finished or want to readd. If you shuffle through a deck without doing anything, you automatically get to the next deck. If it was your first pass, the current deck gets automatically dismissed.

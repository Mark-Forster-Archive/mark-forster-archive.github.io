  • There are many approaches. Here's one: Put all your projects (those you are currently working on and intend to progress) on your AF4 list. Where relevant, put due dates next to the project name.

    Document/plan each of those projects in a separate place. Follow AF to direct you in working on those projects. If some critical tasks come up that you need to do, add such also to your AF list, but only the critical tasks. General tasks are picked up by the project entry.

    Now do the AutoFocus thing with your list. Work on any project you want to, as much as you feel you need to. Rewrite that project at the end of the list. Move on.

  • Here's a tactic I use if I have a hard deadline on a project.

    First, I make sure I include the deadline when I write the task on my AF list:

    - Project A (due 1/31)


    If I am worried the task might get lost for some reason, then I also add a recurring task in Outlook that pops up every day (or every other Monday, or whatever seems to make sense for that project).

    I use Outlook tasks exactly as Mark describes, so I'd write

    - Project A (due 1/31)

    into my AF book every time that Outlook task pops up.

    The process of entering and re-entering makes VERY sure I don't forget about it.

    If it starts cluttering up the AF list, then I might stop entering the new ones.

    If you don't use Outlook, you can do the very same thing using any kind of tickler system, such as a paper-based "43 folders" approach, or a paper calendar, or whatever.

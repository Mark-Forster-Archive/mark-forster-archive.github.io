    Isn't it necessary to have separate lists for separate, well, "contexts", as GTD says? Not that I find GTD useful, but the concept of contexts is true.

    I'd create another list "CITY", where I assemble everything that has to be done there. Am I making things unnecessary complicated? Any comments welcome!

  • Hi Roland

    It sounds as if you may have some confusion over when to move to the next page. You do not have to complete or dismiss all items on a page before moving on to the next. You only have to do one or more tasks in order to move on. It is only if no tasks at all stand out on the *first* pass of any active page that you dismiss all remaining tasks. Therefore, by the time you reach the point of dismissal, you will probably have been to town so many times that the item should be dismissed because you obviously don't really want to buy it or you would have done so! :-)

    You may find some of the posts in the FAQ thread helpful

    http://www.markforster.net/forum/post/645945

  • Roland, if you're interested...

    There is a - paper-based - organizer system that suits the AutoFocus approach perfectly, IMHO. It's called X47 (don't have a link at hand, just google or ask in case of doubt), and I use it in a configuration that consists of 4 parts:

    1. a calendar (one week per double-page: 7 days plus space for time-critical tasks that have to be done in this week, but not on a specific day)

    2. a notebook: which can be used for the AutoFocus "master list"

    3. a "database", a kind of notebook with tabbed sections: which can be used for "grocery lists"

    4. an adress book (where I note not only adresses, but everything else I need to have at hand, like dress sizes, code numbers of printer refill sets, my tax number, opening hours of stores, important customer numbers etc.)

    I used to use it in conjunction with a computer-based project planner, but this feeled always like too much overhead, so I am about to switch to the AutoFocus system, using my X47 in the way described above.

autofocus.cc - Open Source

  • Hello everyone,

    if Mark gives me permission, I will make the next version of autofocus.cc open source. This will allow anyone to install it on his own server or personal computer if they don't want to have their data sitting on a system they're not in control of. It will also make it easier for other developers to contribute.

    I'm using the version control management git for development. The source code will be available here: http://github.com/andreashofmann/autofocus.cc/tree/master

    I'll push the code there as soon as a) the new version is published b) Mark says he's okay with it.

    I'm not sure which license I should use - Creative Commons, BSD-style or GPL. A CC license would be fitting the name AF.cc, but perhaps there are better criteria. ^^

  • I put a draft of the new layout here: http://autofocus.cc/public/data/layout/draft.html

    As you can see, I adopted the "focus on single task" approach I played around with in this thread: http://www.markforster.net/forum/post/776433

    This means you'll strictly follow the af.cc workflow automatically. The mobile interface will always show the current task, the desktop version show more additional information. I kept some things from the last draft at http://autofocus.cc/public/data/layout/draft.html

    Note that the info boxes for due and on notice tasks show how far you are away from them. I hope this will provide more incentive to move through the list to reach those tasks.

    Since everyone has a different monitor size and resolution, I plan to let users choose how many columns they want an which boxes go into which columns.

  • Regarding the license. I prefer the GPL because it ensures that the software I make, and derivates made thereof, will remain free for all times. It makes it impossible to take the source code and release a new product from it to the public without making the source code public as well. It protects the user's freedom rather than the developer's. Many say that BSD-style licenses ate "more free", but it is a philosophical question in my opinion, it depends on what you mean with "free", for example whether to see to the freedoms of the developer who wants my source code, or the user. My vote goes to the user so I use the GPL. For web apps, there is a quite new license called the AGPL (based on the GPL), which might be worth looking up.

    Thanks for a great webapp!

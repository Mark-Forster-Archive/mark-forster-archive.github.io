Latest Variation

  • I've even succeeded in confusing myself with the different variations I've been testing, so I've decided to start a new thread for this one. It combines ideas from several previously tested variations.

    Perhaps my true calling in life was to be a whisky blender!

    ----------------------------------------------

    Current rules for SF (see SuperFocus Rules link in top menu), except:

    Column 2 must be cleared *before* starting on Column 1.

    Urgent tasks are entered on the current page's C2 until you've started on C1. Once you've started on C1, then urgent tasks go on the next page's C2.

    Unfinished tasks always go on the next page's C2.

    Tasks in Column 2 can be deleted and entered on the next page without being worked on. This is the equivalent of putting the task on "snooze".

    However if the next page is the last page, then these "snoozed" tasks are re-entered in Column 1 instead of Column 2.

    ----------------------------------

  • A word or two on what I hope to achieve from this variation:

    The original variation in which it was made compulsory to finish C2 before starting on C1 gave a very tight control and proved to be the most productive of all the variations I've tried. However it was rather a blunt instrument as far as timing was concerned and also the compulsory nature of the tasks started to raise resistance to the system after working with it for some time.

    So to make it easier to adjust the timing I have blended it with the rule about where urgent tasks are entered, together with the "snooze" rule. The "snooze" rule also reduces the compulsory nature of the rules without - I hope - reducing their grip.

    So essentially what is happening is that C1 and C2 form two separate lists which are worked on in turn. C1 tasks are only worked on if they are selected to be worked on, and C2 tasks are always worked on unless they are selected not to be worked on.

  • I've been finding that the rule that "snoozed" tasks can only be entered in C1 on the last page is getting in the way rather than helping. In it's place I'm experimenting with a rule that any C2 task can be returned to C1 at any time.

    So the rules for this variation are now:

    ----------------------------------------------

    Current rules for SF (see SuperFocus Rules link in top menu), except:

    Column 2 must be cleared *before* starting on Column 1.

    Urgent tasks are entered on the current page's C2 until you've started on C1. Once you've started on C1, then urgent tasks go on the next page's C2.

    Unfinished tasks always go on the next page's C2.

    Tasks in C2 may be deleted and re-entered in C2 of the next page without being worked on. This is the equivalent of putting the task on "snooze".

    Tasks in Column 2 may be deleted and entered in Column 1 of the last page without being worked on. This demotes the task back to normal priority.

    ----------------------------------

  • Bernie:

    << I am missing the distinction between the current variation that you described just above on Apr 19, vs. the one you described Apr. 14. In both cases, the feature under test is that C2 tasks can be demoted to C1 at any time. Are there differences in the other details? >>

    There are two differences, one quite major and the other relatively minor.

    1) The ability to put a C2 task "on snooze" has been removed. All tasks in C2 which are not worked on are "demoted" to C1. I was finding that the "snooze" rule resulted in a collection of tasks which travelled along with me in C2 without getting actioned. This removed a lot of the purpose of C2 and also increased my resistance to working on the list. Now the choice is simple: Work on it or it will be put back in C1.

    2) The second change is more subtle. In the April 14 rules the decision to "snooze" a task or move it to C1 was a positive one. This meant that when you considered a C2 task, you had four choices: i) do some work on it, ii) put it on snooze; iii) move it to C1; iv) leave it for now. This was getting a long way from the simple concept of "do it if it stands out". The April 19 rules restore "Do it if it stands out" as the only option. Any C2 tasks which don't stand out are moved to C1 automatically. Although this may seem to be quite a minor change, it makes working the list more intuitive, which was beginning to get lost.

My Project System

  • I've refined this system for a year now. It works with any AF, but especially well with SuperFocus. The system now manages all activity in my life.

    A project is defined as ANY tasks grouped for ANY reason. Besides projects, this covers routines, tickler file, shopping lists, checklists, @waiting, contexts.

    To create a project page, write the title at the top, and tasks down the side. Add notes, diagrams at will. Store these pages in a folder.

    Rules:
    1. Use your AF as normal.
    2. You may take any AF task and put it into a project page.

    SuperFocus only:
    3. At any time you may physically insert a project page into your SuperFocus pages, immediately after the active SF page.
    4. When working a project page, just follow SuperFocus rules. [yes, all of them!]
    5. If you dismiss a project page, move it back to your projects folder.
    Rules for other AF: upon request.

    That's it!

  • I should explain some things:
    1. A project page usually isn't full. So according to SF you may add tasks to the page when you wish.
    2. Many "projects" are recurring, so when you finish a task, you will want to enter it again.
    3. Checklists are a special case. Same rules apply but record things differently: Rather than cross-out and re-enter items, just put a checkmark right of the task it's done. Next time you go through this checklist, add another column of checks.

Prioritizing Projects with Super Focus

  • I've just experienced a strategy for handling high priority projects with Super Focus that I found very effective. I hope others will find it useful. In short:

    1. Write each task associated with the project in your SuperFocus planner as you think of them
    2. Create a reminder for when the project becomes urgent
    3. When the reminder goes off, write down the task "next Project X task"
    4. To action this task, go through your entire SuperFocus planner looking for the next Project X task that stands out. Cross it out and put it in column 2.
    5. When the current Project X task is complete, repeat the process starting at step 4.

    I had been writing down all the things that needed to be done for a project in my SuperFocus planner. The due date was several weeks into the future, so I would work on them whenever they stood out (which wasn't very often, since I had other priorities).

    I also put a reminder that the project would become "urgent" a week or so before its due date. At this point I put the project title in my SuperFocus planner column 2. The activity associated with this was "find next project task". I then thumbed through all my active pages and selected the first activity associated with the project that stood out.

    This task was worked on until it was completed, at which point I repeated the process, going through the list until I found a project item that stood out.

    This allowed me to quickly and thoroughly finish the project with no motivation or procrastination problems, while still fulfilling my other daily duties. Before I had tried this I was concerned that the column two rule of SuperFocus would conflict with looking for through the project "subset" for the next task that needed to be done, but it didn't! By using column two this way, I was able to give my high priority project the time it needed.

    It was also effective at discarding the things on the project that didn't really need to be done. These items didn't stand out as I went through the list looking for the next project item.

  • Ryan, what interests me most about this is how you were able to "add" this "new feature" to SuperFocus without breaking the rules. I put those words in quotes because really the feature was there all along in Mark's rules, waiting to be discovered by someone sufficiently clever.

    If you had suggested to me, "When a project becomes urgent, just flip through the whole book looking for its tasks, and do whichever one you want," I'd have replied, "That will probably work, but you're totally breaking the rules." Yet, all you had to do was enter your devious plan into C2, and just like that, it became legal!

    Digital users will find this scheme especially powerful, since they can search the notebook instantly. It will help if they have consistently included a given project-related keyword in each task.

    This means various ideas such as "ping-ponging" or flagging urgent tasks can be made legit. merely by writing the appropriate instruction into C2. For example, one could write "flag urgent tasks for next 2 days" in C2, and later on, "do one flagged task," or perhaps, "copy one flagged task into C2 below." After all, if we could have pulled these urgencies from any external reminder system—or indeed we could have pulled them out of thin air—then where is the harm in letting our very notebook hold them for us?

    As long as we keep to the active page and continue following all rules about turning to the next active page, dismissing, etc., I don't see a problem.

    Now, when I imagine using your strategy, I have to wonder whether I would tolerate flipping through all those pages. Would it feel too bothersome, looking for a needle in a haystack? Or, would it be perfectly nice? Would I overlook critical things while scanning past all those items? Would my eyes cross after the third page? As Mark often says, you can't know without actually trying it out.

    Luckily, there is a very nice insurance policy: write in C1, "gather all tasks of urgent Project X into a separate list." If all this notebook-searching were to become oppressive, one would expect this C1 task to stand out, and you would end up transferring the project out of the notebook for more focused management.

    But if the notebook-searching went on nicely, the C1 task would not stand out, and so you would press on, leaving well enough alone. Your project would get just the degree of management that it deserved.

    Thanks for sharing that experience, Ryan.

AFH (AutoFocus Hierarchical)

  • It's an idea I've been thinking about for a few days. The idea is to make an expanding list more manageable, and make one think in a "similar, together" way.

    I would be inclined to use software though - this may be beyond what pen and paper can manage.

    So use an outliner program, like ListPro, MLO or OmniFocus, and look at the root level, which might read Work, Home, Play. If you decide to work on Home, move a bookmark line to above Play, move Home to the bottom, and open it up (or in MLO zoom it). This might list Project 1, Project 2, Wash up, Project 3, Feed cat, Project 4. If the cat is noisy you would feed it, so move the bookmark line for this level to above Project 4, and move Feed cat to the bottom of this level.

    The idea is to treat each level as independent AF4 lists, each with an open and closed section. So that if one is working on a work project one sees only relevant things, tidy sock drawer would not show up (the similar together part).

    No I haven't tried it yet, AF4 does seem to be working for me, but it is sub-optimal to have to read through a long, partially crossed out list every time I get to the end of the Open list.

    Has anyone tried something similar?

  • Hi Laurence,

    I'm not sure how similar you think this is, but I was playing on Google Tasks last night (available through my GMail account).

    Tasks allows you to organise a hierarchical outline and to easily move the tasks around the outline. So I was able to play with an outline sorted by importance.

    The feature I liked was the ability to add due dates to any of the tasks in the outline. You can then sort the list by due date. This 'flattens' the list and shows the dated tasks at the top. Within each due date it keeps the ordering of your original outline, so if your outline was sorted by importance, so will your tasks for the day be.

    I was contemplating keeping a master list in the outline together with relevant due dates, then sorting by due date and using an AF-like run through the list top to bottom to see what stands out. Then, AF2-like, I would go to the top of the list again each time I had worked on a task.

    My hope, and reason for posting, is that this would bring open and pressing tasks to the fore. Unless the overhead of maintaining an outline kills the idea before it starts.

  • I've been doing "something" like this from the beginning. But I don't do it in exactly that way.

    I have my life divided up into "areas", which corresponds to your idea. For each "area" I do my planning and pick out tasks that I want to do. Those get dumped into my AF list in no particular order. I don't see any virtue of "doing" by "area", if you get what I mean. IOW, why would I want to do health related tasks all in a row. What I would more likely want to do is all tasks related to a PROJECT, all in a row. So that is what I do. I have "build deck" as the entry on my list, rather than "pick up hammer". That would be too GTD for me ;-) I decide that I want to do deck building tasks based on the weather, my mood, the time of day, etc. Likewise, I have "read" on my list, but don't specify the books. I have those on a reading list and pick from that list. Or if I were to be studying Spanish, I might have "Spanish" on the list and then decide if I am going to learn vocabulary, phrases, or whatever. But I would NOT have a section on my list of "Do fun things". I just don't think that way, I guess ;-)

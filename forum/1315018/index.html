'Completed in a day' AF4

  • Hi everyone,

    Mark wrote (about SuperFocus),

    <<My conclusion from all this is that if you want SuperFocus to work for you, you need to write a much higher proportion of your tasks in such a way that you can finish them in one go.>>

    I wonder if this can work for AF4, with the emphasis on 'completed in a day'?

    I've recently been feeling satisfied about getting some fairly substantial tasks done. However I've not really been following a 'system' as such.

    I believe the recurrent difficulty I have had with all AF versions is that I soon become what I believe Anthony Robbins refers to as 'overchunked', whereby one has many broken down items scattered over a very long list. On the other hand, a project that has not been broken down can be overwhelming leading to procrastination.

    Although it may work against original AF intentions, has anyone tried defining items in a way that they can be 'completed in a day'. If so, how has it worked out?

  • I think it is very important to write tasks in chunks that can be actioned in a day or less. For most of us, if we are building a house (a project not a task) do not really need build a house on a list, we are not going to forget we are building a house. If we have lots of projects we may need a projects list if we are not always working on the tasks.

    For most of the time, I feel it is helpful to focus on tasks and to do so in a manner that causes progress like having them be completed in a time frame we can relate to like a day.

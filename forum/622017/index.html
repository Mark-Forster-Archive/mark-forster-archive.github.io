  • Adrian

    In this case I would put a reminder in Outlook (or whatever system I'm using) to check at a specific date whether I'd received the reports. When the reminder came up I would then enter it in the main task list.

    The relevant section of the instructions is:

    "DO use some means of bringing forward tasks which can only be done at a future date (Tickler file, Outlook reminders, Calendar, etc.)"

    However if a task is only going to be a few days, I wouldn't bother to put a reminder. Instead I'd just leave a task in the main task list: "Bill replied re widget production figures?", "Reports received from team?". Since most pages are active for several days, that can quite happily sit there as an unactioned task until you think it's time to take some action on it..

  • For tracking items needing feedback from others -- if you sent an email originally, I've found it helpful to tag the "Sent Item" with a follow-up flag. I have a "Search Folder" setup that lets me look at all flagged items, and I can simply check that folder once a day to review for any items needing a reminder.

    Couple notes:

    1. I tag the item *after* I've sent it, so the recipient doesn't see the follow-up flag.

    2. I tend to file items right away after sending, but if you leave outgoing items in your "Sent Items" folder, you might try a Search Folder that includes all iitems in your Inbox AND your Sent Items folder, grouped by topic/subject and sorted by date received. Once a reply does come in, the original sent item automatically gets grouped with the reply, so you can see it was flagged for a response and mark the flag complete.

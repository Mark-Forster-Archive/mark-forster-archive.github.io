  • I loved Journler, except for one limitation: like an email program, you *must* sort the main view by some data column. You cannot drag and drop the items into any order you wish. Had it allowed this manual sorting, I would probably still be using it today. Yes, it does have user-defined tags (or Categories or something), which one can rig up to support arbitrary prioritization, but this suffers from the "subdividing tags" problem: what starts out as Level01, Level02, Level03, etc. becomes Level01A, Level01B, Level02AA, Level2AB, etc. as you keep trying to insert items in between others. There was no simple way to automatically renumber everything.

    Does EverNote support arbitrary ordering? How is it for linking items together?

  • I hear ya, Sarah.

    All that linking and tagging and stuff could be pretty handily duplicated in HTML, with the right tool and storage scheme.

    Imagine creating a little blog just for yourself, set to "private access" of course, and you just enter all these notes into it, including file attachments. You can link posts to each other any which way. You can search, yielding a list of hits. You can tag the posts to your heart's content.

    For some people, the "private" checkbox may not provide enough security to put their precious notes online. So, an ideal choice is a blogging tool that stores HTML on your computer's hard drive instead of a web site. Yet, this tool needs to be *simple* to use, not to get in your way when you are trying to work in "idea space." Similarly, we do not want to be coding up HTML tags for formatting or linking—even if readily know how—when we are trying to think.

    Not surprisingly, Journler itself was originally a blogging tool. Evernote looked similar to me, when I read about its features, but if it does not support linking, then a major feature is missing from this perspective.

    There at least seventeen zillion blogging tools out there! Maybe one of them would fit.

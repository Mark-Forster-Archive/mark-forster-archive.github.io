My AF Setup with Things

  • Picture of setup: http://img511.imageshack.us/img511/2540/thingsaf.png

    Summary:

    The "Closed list" project is the closed list used by AF. But I also use it as a place for Most Important Things (MITs) inspired by ZTD, as discussed before here: http://www.markforster.net/forum/post/679222

    I find that putting them in the closed list/backlog helps me finish large, important projects. Basically, I use this as a placeholder for urgent tasks that I would like to focus on that week.

    The "Active Pages" is the where all the todo's would go. This is the "page" used by paper-based AF setups.

    As you can also see, I have three Areas of Responsibilities "Repeating", "Waiting On", and "Dismissed." I reason I use Areas for the last two instead of tags is because it allows me to keep things separated. I fear missing something because some items having the same tag could be near the bottom while others would be on the top. For instance:

    Buy Bony M CD [errands] [waiting on]
    Clean guestroom [home] [waiting on]
    Back up second computer [home] [repeating]
    Organize magazines [home]
    Post setup on Mark Forster's forum [computer]
    Clean software box in bedroom [home]
    Week 1 of Algebra 1 practice [home]
    Shovel driveway [waiting on]

    Although the above is a short list, I still see how I could procrastinate and miss something. This mess of tags and contexts is one of the reasons why I decided to leave GTD for AF in the first place.

    For repeating items, Things has a limitation that prevents items from being repeated in a Project. So, I cannot put, say, "Renew pool membership" inside my "Active Pages" Project. Instead, I have to use the Repeating Area. Although this may seem like a limitation at first, I actually like how recurring events do not clutter my Projects' screen.


    Some basic questions answered:


    Question 1: Why use electronic methods instead of paper-and-pen, like Mark Forster recommends?

    While I liked using a notebook (I used a small moleskin previously) for AF, I found that it often got pretty messy when I used it. It was also difficult to keep track of recurring events as it would clutter up the pages. Also, when I went out I had to carry another notebook with me to write things on (like a grocery list), so I had to keep switching them back and forth.

    But more importantly, although I loved the workflow of AF, my moleskine did not *feel* like a safe system that I could rely on. Now I only use my notebook for notes and brainstorming.


    Question 2: Why use Things for AF instead of similar software like The Hit List or Omnifocus?

    Firstly, although Things is not really recommended for a GTD-like implementation, I feel it is the best for something like AF. Its limitations actually become benefits when using it with AF.

    In the beginning, I used Omnifocus to try to implement AF, as I had great success with the program with GTD. However, I felt the program was too complex and clunky for me and seemed to ignore the whole feeling of "free-flowing" productivity that AF allowed with a notebook.

    I then tried to use The Hit List, because someone else used it with AF (http://www.markforster.net/forum/post/740814#post749820). I tried to use it, but I found that I was tweaking it more than actually using it.

    Finally, I decided to use Things (which I unsuccessfully tried before for GTD). Immediately, I found the interface an actual pleasure to work with, and its simplicity allowed the right mix of AF's workflow and some structure to keep things from falling apart.

    Question 3: Why not use multiple Projects like "Page 1," "Page 2," "Page 3" instead of
    one large "Active Pages"?

    Although using multiple pages makes sense when using AF with paper-and-pen, it is not entirely necessary with electronic methods. Since Things is able to hold large amounts of items in a Project, I really had no need to divide them up into multiple Projects.

    Personally, I like how everything is in one place in this setup, as I am able to preview the whole list without having doubts of missing something that might be on another page. Of course, this might change over time, but at this moment I do not see the need.



    I'd really like to thank Mark for creating this system; I finally, after many months of using GTD, have found the "holy grail" that I was looking for. By using Things, I also feel that I eliminated the feeling of something 'missing' with AF on long-term goals, compared to DIT or GTD. It works like a complete system for me now. :-)

    P.S. I apologize if my list of todo's are small; I actually had a lots of things to do, but I took the screenshot after completing most of them. (See?! That proves that AF works!) :-P

    P.P.S If anyone has any questions regarding my setup, feel free to ask!

  • The 'Someday' section is for ideas or things you might want to do in the future. Basically, it acts as a placeholder for ideas you are interested in but cannot do immediately. For instance, "Build outdoor swimming pool" is an idea you want to pursue in the future, but not right now since the weather might be too cold or you might have other priorities.

    I review this section weekly, and I drag-and-drop items in my active list if I want to start working on them.

    I used to write my ideas on index cards a while back, but I often misplaced them and forgot about them altogether. :-P

    To clarify the other sections:

    'Inbox': This section follows GTD's philosophy of 'capturing everything in one place'. But since using AF, it has become redundant, as I now add items directly to my "Pages" Project. So, I use this section as a placeholder for incomplete items, commitments, or goals. This is essentially the "Think More About" section.

    'Today': This section acts only as a reminder for due items (like "Return library books"). Any event scheduled for today is added automatically to the 'Today' section. This prevents me from missing a deadline.

    'Next': This view shows all "Pages" at one glance; I use this to review my active lists.

    'Scheduled': Items with a hard deadline get moved here. I then create my calendar by reviewing this section every week.

    'People': This section is for delegating work to someone else.



    So all in all, this setup is working very well for me. :-)

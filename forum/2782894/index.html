  • You should definitely try Zim-Wiki. See https://zim-wiki.org/ and https://github.com/zim-desktop-wiki/zim-desktop-wiki/. It's quite similar to Obsidian, but faster and more mature.

    I have a dedicated page for a FVP master list. Each entry has a checkbox that can have 3 states (indicated either by a checkmark, cross or arrow forward). For dotting I use "arrow forward". For placing existing tasks to the bottom I either copy and paste them, or use the "Line Sorter" Plugin, to move them down, line by line, with a keyboard shortcut, see https://zim-wiki.org/manual/Plugins/Line_Sorter.html. This is a default plugin that you have to enable in "Options > Plugins", if not enabled already.

    For complex projects I link tasks from the master list to another page with detailed step-by-step tasks. This keeps the master list decluttered.

    All content is stored in plain text files and can be easily read and manipulated by other apps. My preference on Android is Markor. See https://play.google.com/store/apps/details?id=net.gsantner.markor and https://github.com/gsantner/markor. Don't forget to select the Zim-Wiki format, when reading Zim-Wiki pages in Markor.

    Enjoy!

  • I found this old thread because it is suddenly very relevant to me. I've been a Dynalist user for years, but I've left it and now I'm all in on Obsidian. I don't use it as a Zettlekasten in the purist sense but the ability to easily create and relate note files is amazing.

    Here is how I've been using it with FVP principles and sometimes randomization.. I create a separate note file for each project. I also have a note file for each person I regularly meet with. Obsidian excels at linking notes together with hyperlinks so I can easily link project files with people files. My job as an IT manager is to work with people and projects. For the one-off tasks that only I can do, I put those in a note file with my name as the file name. I currently have about 50 active people and project note files for work.

    The way Obsidian displays the list of files makes it easy to read through the file names like a list. All the files are listed vertically down the left side in the app. Obsidian allows you to sort this "list" by last modified date. Keeping the oldest modified file on top ensures that I don't let any project or person go too long before checking in on it/them. I can treat this list of files like a FVP list. The way I select the file is to simply open it and it opens a tab within Obsidian. I make sure I right click and chose "open in new tab".

    Per the FVP rules the first item on the list is always selected first so this is my oldest file. (I only do this in my "active files" folder.) The opened files open as tabs horizontally across the top in Obsidian. Sort of like a web browser. In FVP fashion, I can keep going down the list asking the question and open the file that I want to work on before the last selected file. Then per the FVP rules, I can then work the tabs right to left once I have all the selected files open. Right to left is because these tabs stack horizontally as I open them. To show I actioned something about the project or person file, I simply update the file with a relevant note. Then I can close it. This effectively moves the file to the bottom of the list because it now has the most recent modified date. If the project is done, I can delete the file entirely or move it to another folder. I only use FVP on the "active files" folder. Within each note file I often have a list there as well but I usually use my intuition on the tasks that might be within the file.

    I suspect you could do this method with any text editor and a filing system but the
    Obsidian app has a nice layout that seems to keep it all together nicely and I'm really becoming quite fond of markdown syntax. Obsidian also has a cool open random note feature that I often use when I'm feeling indecisive.

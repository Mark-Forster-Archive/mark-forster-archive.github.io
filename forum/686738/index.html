Spreadsheet AF list

  • Though I'm using paper for my personal list (a very portable pocket size Moleskin cahier) I'm currently trying a Google spreadsheet for my work list. As a software developer my working life is heavily computer-centric so this suits my workflow.

    Has anyone else tried the spreadsheet route?

    For me, a spreadsheet has more of the flexibility and immediacy of paper than other digital solutions and has the powerful advantage over paper of easy copy and paste and input by typing (I can type much faster than I can write neatly).

    For example for the task 'check out this web article' I can paste the URL into the task notes column and then click on it to open the link when doing the task. It's also reasonably easy to view/filter and edit from the iPhone web browser. Doesn't yet support offline access via Gears but I wouldn't be surprised if it soon does.

    Currently I have each 'page' on a separate sheet with a single character status column for started/in progress/dismissed/finished which is automatically coloured making it easy to spot the unprocessed tasks. It's easy to add extra columns as needed (e.g. notes), copy and paste items, sort tasks, print out, import, export, share with other people etc. Creating new empty 'pages' is a matter of duplicating an active sheet and deleting the existing items. I usually name the sheet with its date and time.

    The main limitation with a Google spreadsheet is that rewriting an item at the end of the list requires a copy and paste as there are no macros. The easiest way to deal with this is to have one browser tab for the current 'page' and a separate browser tab for the last 'page'.

  • Based on my personal experience, I would start with a notebook in paper form. I used a spreadsheet system with GTD and I think part of the reason why I wasn't very successful with that system was the fact that I relied on various digital solutions for it, including using a spreadsheet.

    A spreadsheet has all the advantages and benefits that you mention, but it also has a serious drawback: It distracts one from actually doing the tasks into trying to perfect the system. Moreover, if you print your spreadsheet in order to use the paper list as a working list, then every update you do to the list during the day must also be entered in the spreadsheet in order to update that one too. I find that there is a break in continuity. In any case, looking at a list of tasks on a monitor is not as inspiring as looking at it in hard copy format.

AF2: Implementation Idea

  • What do you (and esp. Mark) think about the following paper based implementation?

    When starting a new book, one does not write the first task on the first line of the first page but instead on the last line of the last page. New tasks are then added filling the book backwards. AF2 is then implemented as described by Mark, just the other way around:
    - the last block of unactioned items is set "on notice"
    - the initial scan of the list is done back-to-front
    - working with the list is done front-to-back (which seems to be more natural for most of us western people), so you always start from and return to the very first item of the list.

    I haven't tried it yet (I still have a couple of empty pages in my book), but I guess I'll give it a whirl... From my perspective all the AF2 rules are adhered to.

  • My AF2 implementation uses index cards, one item per card. The "list" resides in a small box with the newest item at the front. I found these advantages:
    - It seems "natural" to add new items at the front.
    - It seems "natural" to work through the box from front to back.
    - Handling the cards takes a little time and causes me to think better about the item. I found that with a simple list, I read through the items too fast.
    - It is possible to add further notes to each item easily.
    This is the system I am most happy with, from all I have tried up to now...

    the completed items are removed. Some of them are recurring items, they are put back into the tickler (I'm very proud of my index card tickler box).

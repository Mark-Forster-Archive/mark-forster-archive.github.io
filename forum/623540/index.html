Which list?

  • I've realised that there is ambiguity the instruction "Cross the item off the list, and re-enter it at the end of the list if you haven't finished it"

    I've been putting it at the end of my long multi-page list, consistent with the idea that, though it might be spread over several pages, there is only one list. "The system consists of one long list of everything."

    The problem with doing this is that an item at the start of a 100-item list will be reentered 100 items further down, whereas an item near the end of such a list will reentered just a few items further down, and revisited almost immediately.

    An alternative approach is to reenter the item on the same page as it is already written. This is consistent with "You work through the list one page at a time ". It doesn't suffer from the shortcoming just identified. It also avoids lots of page flipping, and where a task is done a little bit at a time the fact will be obvious as the one page will have it crossed out and rewritten repeatedly. The downside is that one might run out of room on a given page.

    Which approach did you have in mind?

  • David:

    Frank is right. The re-entry goes at the end of the whole list. This may be 100 items down the line, but once you get there and work on the item again the re-entry will probably be much closer, so it will average out.

    If you put the item on the same page, and then move onto the next page, the re-entry is still 100 items away, so I don't see the advantage.

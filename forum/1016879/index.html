DWM Electronically -- the K.I.S.S. approach

  • Hi all,

    I've been using DWM very successfully for about 12 days now, and absolutely love it. My implementation is 100% electronic, but I seem to be doing it differently from the others running DWM electronically, so I thought I'd post here and see if this helps anyone (or if anyone can suggest ways I could be doing it better).

    It sounds to me as if everyone using DWM in an electronic form is making use of "date due" entries for each task, and somehow grouping the tasks by date. This seems like overkill to me -- I tried this, and found it too complicated for my taste. So I came up with a much simpler approach. In fact, I deliberately used a free but very simplistic To-Do list application for the iPhone (it's called SimpleList Free -- ad supported but I never even notice the ads even though I'm using it constantly) so that I wouldn't get distracted by any bells and whistles. Basically, this approach can be used in *any* to-do list program that allows you to have more than one list.

    What I've done is setup a bunch of To-Do lists, where I use the date as the name for the list. So, for example, my current lists look like this (it's the 17th of February today in New Zealand):

    2010-02-17
    2010-02-18
    ...
    2010-02-23
    2010-02-24 REPEAT
    2010-02-25
    ...
    2010-03-16
    2010-03-17 NEW

    Each of these is a separate To-Do list, where (just like a calendar) each list contains the items entered for that day.

    I spent about twenty minutes setting up all these lists at the start, and they've worked brilliantly for me. If I click on a list, I can drill down to see the items for that day. SimpleList has the advantage of showing how many unchecked items there are in each list, so I can see at a glance which pages have uncompleted items on them, and how many items I have on each date. Also, the application's badge shows the total number of items across all pages, which is a helpful indication of how much my lists have grown.

    Note that I used the "YYYY-MM-DD" coding scheme for dates. That way, I can sort the lists alphabetically and they appear in the right order. I also found it helpful to add "REPEAT" and "NEW" beside the "one week from today" and "one month from today" lists, so that I could quickly find the right list if I wanted to add or re-add an item.

    Each morning, I add an extra list to the end, delete yesterday's list (no dismissed items yet -- hurray!), and rename the "NEW" and "REPEAT" lists. This takes me maybe 30 seconds, and sets me up nicely for the day. I also review my lists at the same time, to see what I need to focus on for the day.

    Frankly, Mark's new system is absolutely fantastic. I had mixed success with AF...it would work for a while, but the lists would grow stale after a while. With DWM and its "conveyor belt of doom" (as I like to call it), there's no chance of the lists growing stale. I've been incredibly productive ever since I started using it, and can't ever see myself looking back.

    Anyway, I just wanted to suggest a simpler way of running DWM in an electronic form. You don't need to be fancy (in fact, I bought OmniFocus to see if I could/should use it instead of SimpleList, but it seems to make things too complex). I like the simplicity of my multi-list approach, and I suspect others may find it useful too.

    There are two things I'd love to see that I can't do with SimpleList. Firstly, it'd be great to be able to flick back and forth between the lists, a la FocusPad. And I'd like the ability to "flag" some items and then show a separate list of just the flagged items (I'd use this for identifying all the items to do while running errands -- I found myself writing a separate list on paper when I had to go into town).

    If I could use FocusPad or OmniFocus or whatever to do these two things, I'd be in hog heaven. But even with the simple system I've got, I'm more than happy -- the K.I.S.S. approach lets me focus on the process of running DWM, rather than getting bogged down in the implementation.

    Cheers,

    - Erik.

    I believe the key question to ask is **complicated** for whom/what?
    Is it complicated for the user to use?
    Or is it complicated for the electronic form/list/program to automate?

    I'm doing DWM electronically using AndreasH's brilliant AutofocusAHK (for Windows only though).

    AFAHK is very easy to use for DWM. Basically only the following three steps ...
    (1) You add new tasks by pressing Capslock-A and enter task description. AFAHK will choose the correct date to put it in. You can add more tasks at any time.
    (2) You start working your DWM list by pressing Capslock-D Do Work. AFAHK will automatically present the next task for you to decide to start or skip to next task.
    (3) After starting and stopping work on a task, you can re-add the task, and AFAHK will automatically choose the correct date to put it in. AFAHK will then present the next task for you.

    Internally AFAHK is complicated with 4 different dates/times for each task. It tracks when you add the task, when it expires, when you do it, how long you do it. Those info is beautifully exported into an HTML page. You don't need to learn the complexities, AndreasH have done (and is doing it) for us, who DWM in front of a Windows PC most of the day. ;-)

  • Hi Erik:

    As someone who’s very recently posted about implementing DWM in a variety of apps using a due date system, and given what the second ‘S’ stands for in ‘K.I.S.S.’, it would be possible to take offence. Instead, I’ll just point out that the ‘K.I.S.’ part seems misdirected. Implementing the core of DWM in The Hit List is *much* simpler than the system you describe. Set up takes less than a minute (and so a lot less than 20 minutes): start a new list (your DWM list), set it to display due dates only, and set it to sort by due date – that’s it. Assigning dates to tasks is then extremely simple, too: after entering the name of the task in the usual way, hit tab once (to be in the “Due” column for that task), and type either ‘1m’ or ‘1w’ as appropriate. Hit return and you’re done. THL translates those bits of shorthand into dates, and automatically sorts tasks by date, given the set-up. What’s more, everything’s now on one list, so no moving between lists all the time. (And you can archive any completed tasks with a single keystroke to keep the length of the list down. If you want to see the tasks you’ve archived, back in their original locations, that’s easy, too – there’s a keyboard shortcut – as is hiding them again.)

    Other pros and cons are another matter, and this is a desktop app rather than an iPhone app, but if simplicity is the crucial issue....

  • I see Rick Kotter is also implementing SFv3 in Toodledo. I wonder how he do it. Ref http://www.markforster.net/blog/2011/2/20/some-suggestions-for-what-next.html#comment11975498 .

    Anyway, I've just started moving my Toodledo DWM list to SFv3 ..
    (1) I use folders ".01", ".02" etc for pages. Sorting the folders alphabetically allows my old folders used for project names to be sorted to the bottom.
    (2) I set default for new tasks to have the last page folder number
    (3) I use priority for columns, and will also use "negative" priority for dismissed tasks, that can be easily hidden.
    (4) My screen shows about 18 tasks, so I decided to use that page length for now.
    (5) My default view is by "Folder", sorted by Folder|Forwards and Priority|Reverse, because I believe usually only column 1 fills up (not column 2).
    (6) "Folder" view "All Tasks" tab and "Close Dividers", can give an overview numbers of tasks and those done for each page.

    I did consider using tag for pageno instead of folder. But I didn't see much pros/cons for either tag/folder, so decided for the latter. Is there a clear advantage for using tags for pageno, Will?

  • Further feedback on my SFv3 on TDD ...

    I tried that Folder view, sorted by Folder|Forwards and Priority|Reverse for some time. But I find that the tasks are not consistently being sorted the same way. Without changing any task, a reload of the page gives a different sort. That bugged me.

    I then tried Folder view, sorted by Folder|Forwards and DateModified|Forwards. Unfortunately that also didn't work well enough because it's really DateModified only, not Date+TimeModified.

    I finally settled with Folder view, sorted by Folder|Forwards and Tag|Forwards. Folders are "01", "01a", "02", "02a" etc, where "01a" is Column 2 for page 01. Tags are "01", "02", "03" etc, keeping the task number for the page. I keep max 25 tasks per page. Since I hide completed tasks, this limit is really 25 "not-yet-done" tasks, that I hope would not adversely impact SFv3 flow.

    Mmm, is there a better way of implementing SFv3 on TDD?

DWM: Reviewing dismissed tasks

  • Hi,

    Since I first read DWM's specification, I wondered why there's no reviewing, like AF1's.

    Lately I've been quite lazy with the list, and let several tasks die. I'm fine with that, sometimes even happy to see things vanish, even if they are important.

    But I suspect that I'm abusing the method: I don't do things that bother me, because if I let them alone for a month, they'll just go away, I won't see them again.

    I don't know if Mark purposely excluded this from DWM, or if the specifications aren't final, but I think I'm going to do some reviewing from time to time. Rephrasing/spitting a task was quite effective with AF.

    I'm using a digital implementation, so expired tasks are always there for reviewing.

    Thoughts?

  • I've been using MLO since Oct. 2007 for DIT. On day 1 of the AF betatest I tried to start on paper but as my handwriting is rather spidery (Ger: krakelig) I soon fell back to a computer implementation. First I tried Excel because I thought here I can see the number of lines I use on each tab (= page). But in Excel I missed the automatism I had with MLO and DIT like my tickler list or regularly recurring task (e.g. my Monday, Tuesday etc. or weekly and monthly lists) which automatically "pop up" in my MLO file when their time is due. I then had a "flash of insight" how I could simulate the 30 lines per page mechanism in MLO.

    - Autofocus
    -- page 10 23.01. ===========
    ---- task 1
    ---- task 2
    ...
    ---- task 30

    -- page 11 24.01. ===========
    ---- task 1
    ---- task 2
    ...
    ---- task 30


    - tickler
    ---- future task x on 20.2.2009 (MLO will automatically show me these items on that date)
    ---- future task y on 12.3.2009


    - recurring tasks
    -- weekly
    --- ® copy Monday tasks to AF on 16.2.2009 + recurring every week (tasks are listed in the notes field)
    --- ® copy Monday tasks to AF on 17.2.2009 + recurring every week
    ... and so on ...
    -- monthly
    --- ® monthly backup on 1.3.2009 + recurring every month
    --- ® Review dismissed items on 10.3.2009 + recurring every month


    - Projects
    ...


    - Dismissed (here I copy those tasks which I dismissed from Autofocus above => easy reviewing in one place)
    ...


    There is unfortunately still a manual process in my MLO implementation of AF because I don't know how to automatically get the current number of subelements (childs) for the "page" elements above. So my workaround is to use 2 additional helper elements named "____ (10) ____" and "____ (20) ____" where I manually count the lines on a page after I added a certain amount of tasks and put these helper lines after 10 and 20 tasks so that I then can easily spot when I've reached 30 lines and must create a new page. In case I have added more than 30 lines on a page the new MLO version can now move all the surplus tasks to the new page after it is created (CTRL+M).

    The beauty with MLO is that with SHIFT+CTRL+M you can always add new tasks to the end of the current last page independent of which other application has the focus at the moment as long as MLO is running in the background.

    I am totally happy with my 2 MLO files (work + home) as I have all important things in one place.
    Every day I first start with an MLO view where I check whether new tasks got activated from tickler or recurring tasks and copy (recurring) or move (tickler) them to the last page in AF. Then I continue with my AF view where I can see all active pages and can easily collapse (cursor left) or show (cursor right) the tasks of 1 page.

    Hope that helps,
    Klaus

  • The second "® copy Monday tasks to AF on ..." above should read "® copy Tuesday tasks to AF on ...".

    In the notes field of such a task I have listed all the task I want to do on that day (or shortly afterwards). For example
    ® Inbox Zero
    ® archive papers

    I then only have to do the following sequence:
    CTRL-A (= "select all" tasks in the notes field)
    CTRL-C (= copy selection to the clipboard)
    SHIFT+CTRL-M (= open rapid task entry window of MLO)
    CTRL-V (= paste clipboard contents into MLO window)
    click "Add tasks" button

    => now all new tasks are in my AF list at the end of the last page and I don't have to retype them or anything like that.

    I use the ® as a marker for me that it is a recurring (routine) task. Thus I see at once in the AF view that the task is well known and does'nt require extra (unknown) effort to get started or finished.

  • Hi Tigerigis

    I work mostly in the To-Do view. I set up a filter for Autofocus like this:

    Group by
    1. Top Level Parent
    This group expanded, Descending

    2. Parent, Descending

    x Show items count in groups

    This last option is very cool because I see the number of open tasks per page at the end of the line at a glance.


    In To-Do I have my pages collapsed and visible as one line each.

    + Autofocus (5)
    .. + 11 24.01.2009 ============= (3)
    .. + 12 27.01.2009 ============= (2)
    .. + 13 01.02.2009 ============= (11)
    .. + 14 07.02.2009 ============= (14)
    .. + 15 21.02.2009 ============= (9)


    I have only 1 context for the whole Autofocus subtree. I then select a page (e.g. 12 above) and press cursor right so that this page expands and all tasks of it are visible. I work on this page like AF says and when I am done I press cursor left and collapse this page again and continue in the same way with the next one.
    I love this way of working because I am not distracted in the To-Do view by already finished and striked out lines.

  • Hi JD,

    I created 5 different views:

    1.) "feed into Autofocus"
    Shows all tasks which are active from "tickler" or "recurring tasks (routine)".
    No Autofocus tasks are shown in this view. I start with this view every day and empty it. But I do not do any action I only decide for each of those new active tasks whether to move / copy it into the AF subtree now or delay the task for some time, because it is not yet urgent or import enough to have it in my AF list now. I don't want to overload my AF list with too many active tasks, because this demotivates me.

    2.) sort by Due Date - this month
    Shows which tasks are overdue, due today or due soon from Autofocus, tickler and routine. This gives me a short overview about urgent tasks.

    3.) Goals
    Shows which tasks are marked in MLO as weekly or monthly goals. This gives me a short overview about important tasks. I prefer to mark important goals as monthly goals because you don't see this later in the Autofocus view and they thus do not stand out.

    4.) Number of tasks in Autofocus
    I just have a very short look how many tasks are open overall in my AF list. Use "Group by" "Top Level Parent" only and set "Show items count in groups"

    5.) Autofocus ##### open tasks
    This is the view I use to do my AF in.

    JD> The drawback I suppose is that every top level parent shows up, but when collapsed, it doesn't distract you from your AF pages.

    You can select the contexts which should be displayed in a specific view, so I do not get distracted, because I only show the context "Autofocus" (contains all my Autofocus tasks) and and 1 additional context ("urgent today").

    This "urgent today" context has the same effect as the post it notes described by some of the "paper using people" => I see at the top of the To-Do list those special tasks which have to be done today and can thus integrate them into the "scan" of a page to decide what task stands out. I only used this "post it" feature once or twice though, because I usually do not get such important tasks with such a short timeline. And when I do, well, then AF has to wait.

    Hope that helps,
